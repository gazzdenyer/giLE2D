Your Map Backdrop!
 
OK, in this section, we are going to learn how to add backdrops to your map.
Since there are two types of map by default that we can create, static and scrolling, we will pick the more involved backdrop, which is scrolling. But first a note on static maps.

A static map will generally have only one backdrop. Therefore, if you pick this option when you create your map (see Creating Your First Map) then the Back Drop dialog, pictured below, will only have the first slot open. This does not mean to say that you are limited, though. It entirely depends on how you code your game.
You could have chosen the scrolling map option when creating your map, but your game code dictates entirely how you utilize these resources!

Anyway, go ahead and create a scrolling map (if you have not done so already).
As you know, giLE2D comes with some cheap artwork! So, for this example, I'm going to use it!
Now, with infinite enthusiasm... do the following.

1. Click on the "Back Drops" button, shown below. The "Edit/Change Back Drop Files" dialog will appear to you.
 
 
In this dialog below, you will see four slots. giLE2D supports up to four layered backdrops for your maps. The key here is to already have your backdrop files ready.
In this example, I am only going to use three backdrop files, since the scroller framework suppied with giLE2D is coded for only three. But, as usual, you are free to modify this code for your own devices!

Anyway, I recommend that you start with the background file (like a sky, etc... something thats going to look like its in the far distance).
 
 
To add the first file, double-click on the first slot. A "Load Background" dialog will appear. Simply navigate to where you keep all your map media, and choose your far-distance backdrop file. In this case, I am using a nice space themed PNG file called "back1.png".

NOTE: It is necessary for these files to be 1136 pixels in width, so that the image can be repeated correctly by the framework code. Again, you can always code this to your liking, but for the example framework, make sure it is 1136px wide. A netter explanation of this is given here.
 
 
To add the second file, double-click on the second slot. Again, a "Load Background" dialog will appear. Simply navigate to where you keep all your map media, and choose your mid-distance backdrop file. In this case, I am using a nice mountain range themed PNG file called "range1.png". It has an alpha channel in it that will allow me to see the far-distance backdrop through it.
 
Finally, to add the third file, double-click on the third slot. Again, a "Load Background" dialog will appear. Simply navigate to where you keep all your map media, and choose your foreground-distance backdrop file. In this case, I am using a nice grassy-tree-ish range themed PNG file called "ground1.png". It has an alpha channel in it that will allow me to see both the far-distance and the mid-distance backdrop through it.
 
 
You will now see a nice little preview of how the combined backdrop will actually look (my programmer art notwithstanding).
 
 
If you are happy, or just plain impatient, go ahead and click "OK". The backdrop will now be rendered in the main giLE2D editor.
Note that it appears stretched! Well, thats because we did not go to the length of tiling the images across the repeater multiplier that you chose when creating your map. It keeps things less complicated resource wise.
When you get to the point of actually running the map in the example framework supplies, you will see the backdrops scrolling by at different rates, all correctly tiled and repeated. This is parallax scrolling at work!
