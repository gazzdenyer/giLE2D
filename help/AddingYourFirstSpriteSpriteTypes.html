Adding Your First Sprite - Sprite Types
 
 
Alrighty then, its high time we start getting some stuff into our map... isn't it... or is it?
First, I'd really like to tell you about Sprite Types. Its kinda important.
If you take a look near the top left of giLE2D, you will see a drop down called "Sprite Type To Create". This is important and somewhat cool.
 
 
We can see from this list that there are nine pre-defined sprite types that are incorporated into giLE2D. Later on, we will discover that we can create an ENORMOUS number of different sprite types, by adjusting the 31 properties that each sprite has, but for now, this helps us quickly add sprites with pre-defined properties. This really helps speed map development up!

Anyway, I will list each pre-defined sprite type, and tell you just exactly what the definition of each type is...
 
1-Map Geometry This sprite type defines anything in the map that is static, and can be assumed to be a platform, ground or other rigid body
2-Collectable Item This sprite type can be picked up by the player, embuing said player with points, health, money or whatever you code it for!
3-Scenery Block This sprite type is part of the map scenery, similar to type 1, but you can "pass over" or "through" it. Think trees, rocks, signs, etc.
4-Enemy Entity A sprite type that is your enemy! Can be configured to do damage. Its collision is set so that if the player touches it, damage is done! Think lava, water, critters, prickly cactus, pools of horse manure, etc.
5-Friendly Entity A sprite type that is your friend! Can be configured to help you! Depending on how you code, it can follow you, give you manna, be your surrogate mother, etc.
6-Moving Platform The sprite type can be configured to move around the screen, given minimum and maximum x and y coordinates. Surely an elevating experience! The player can jump onto them and feel uplifted!
7-SpringBoard BOOOIIINNG! This sprite can be used as a jump pad! Maximum height can be configured, along with collision dynamics.
8-Teleporter Beam me up (or across, or down) Scotty! This sprite takes destination x and y coordinates, and transports the player there! See the example frameworks for example code.
9-Player Start The most important sprite type. This is a marker that defines where your player will start on the map (just make sure you use an appropriate representation!)
 
When we eventually go to add a sprite to the map, we will pick one of these pre-defined sprite types. It will help us to quickly create our prototypes.
But, there is another thing that we need to be aware of before adding sprites... and that is the use of LAYERS.
Layers will allow us to code for stuff like "should the player pass in front of, or behind this doo-hickey..." type thing.

Layers allow us to control the rendering, or "Z-Order" of the sprites. It also allows us to group like-sprites together!
