Adding Your First Sprite - The Layer Manager
 
 
Well then, its time that we set up our map layers so that we can organize our sprites better.

Layers are good because they allow us to code for groups of sprites that do the same thing (e.g. ground sprites, collectables, etc) and it also allows us to control the Z-Order in which they are rendered.

With AGK as an example, it supports almost 10,000 layers. That means that sprite 1 can be behind sprite 2, which can be behind sprite 3, etc, etc.

Although its not necessary to to so in giLE2D, its always great practice to define to yourself what types of objects are going to appear in your map up-front, before creating anything.

With this in mind, lets have a look at giLE2D's Layer Manager.
 
The Layer Manager
 
First, click the "layer Manager" tab, as shown on the image to the left here.

You will see a toolbar with four buttons on it, and a white-space which will show the list of layers that you have created/will create. The image on the left is a representation of "here's one I did earlier..."

Lets say, for the sake of protracted gibbering, we want to have five layers in our map, because we have five discreet types of sprite that we want to have.

These discreet types are:

Map Geometry
Liquids
Collectables
Player (most important!)
SpringBoard

When we go to actually add a sprite to the map, we are going to first select the sprite type (see here), and then choose the layer that the sprite belongs to.

Anyway, before we get ahead of ourselves, lets add a layer.

1. Click the "Add Layer" button
2. An "Add Layer" dialog will appear for you.



3. Fill in the name of the layer you wish to create (in this case, "map Geometry").
4. Click OK... the layer will now appear in the layer manager list for you to use.
5. Repeat steps 1 - 4 for each of the other types (Liquids, Collectables,Player, SpringBoard)
6. Note that each layer is given an ID that you can manipulate in your game code.
 
So, now we have our map layers.

Note that we can also delete layers or even change their order

Deleting layers has a number of considerations that need to be looked at, especially when you already have a bunch of sprites in your map that belong to the layer that you wish to delete. However, that is a topic for later discussion.
 
So then, we know how to create layers...Are we getting nearer to adding an actual sprite to the map? Oh, yes, we are... oh, yes.
